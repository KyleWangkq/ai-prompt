# 全局词汇表 (Global Glossary)

## 核心业务术语 (Core Business Terms)

| 中文术语 | 英文术语 | 定义 (Definition) | 业务含义 (Business Meaning) |
|---------|----------|-------------------|---------------------------|
| 支付单 | Payment Order | 系统内生成的支付请求记录，包含支付金额、类型、状态等信息的业务单据 | 连接订单与支付的核心载体，承载完整的支付业务信息，是支付处理的基础单位 |
| 支付单号 | Payment Order ID | 由支付模块生成和管理的唯一标识符，用于区分不同的支付请求 | 系统内支付业务的唯一追踪标识，确保支付记录的可追溯性和数据一致性 |
| 支付渠道 | Payment Channel | 执行实际支付操作的通道，包括第三方支付服务商和企业内部支付系统 | 支付执行的具体路径，决定支付方式、处理时间和费用结构 |
| 交易号 | Transaction ID | 支付渠道返回的唯一交易编号，用于在渠道侧标识具体的支付操作 | 支付渠道侧的交易凭证，用于对账、查询和问题追踪的关键标识 |
| 支付流水 | Payment Record | 记录每次具体支付操作的详细信息，包括渠道、金额、时间等 | 支付操作的完整记录，用于退款选择、对账验证和业务审计 |
| 合并支付 | Batch Payment | 将多个支付单通过同一支付渠道进行统一支付处理的业务操作 | 提高支付效率，降低手续费成本，简化企业财务处理流程 |
| 分批支付 | Installment Payment | 将单个或多个支付单分多次进行支付处理的业务模式 | 适应企业资金安排，支持灵活的支付计划，降低单次支付压力 |
| 回调 | Callback/Notify | 支付渠道在支付处理完成后主动通知系统的异步通信机制 | 确保支付结果的及时同步，保证系统状态与实际支付状态的一致性 |
| 企业用户 | Corporate User | 在系统中注册的企业实体，作为B2B交易的参与主体，具有支付资格和权限 | 支付业务的主要执行者，决定支付方式选择和支付计划安排 |

## 领域特定术语 (Domain Specific Terms)

| 中文术语 | 英文术语 | 定义 (Definition) | 业务含义 (Business Meaning) |
|---------|----------|-------------------|---------------------------|
| 预付款 | Advance Payment | 在设备生产或服务提供前，客户支付的首期款项 | 确保订单有效性，为生产方提供启动资金，降低业务风险 |
| 尾款 | Final Payment | 在设备交付或服务完成后，客户支付的最终款项 | 完成交易闭环，确保服务质量后的最终资金结算 |
| 其他费用 | Miscellaneous Fee | 除预付款和尾款外的其他相关费用，如运输费、安装费等 | 覆盖交易过程中的额外成本，确保费用结算的完整性 |
| 支付单类型 | Payment Type | 区分支付单业务性质的分类标识，用于业务处理和财务核算 | 支持不同支付场景的差异化处理，便于业务分析和财务管理 |
| 企业内部账户 | Corporate Internal Account | 企业在系统内管理的资金账户，用于内部资金调配和支付 | 提供企业资金的统一管理，支持快速支付和资金优化配置 |
| 信用赊账 | Credit Payment | 基于企业信用额度的延期付款方式，支持先服务后付款 | 提升交易效率，支持信用良好企业的灵活付款安排 |
| 线下转账 | Offline Transfer | 通过银行等金融机构进行的传统转账方式，需要凭证确认 | 适应传统企业支付习惯，支持大额支付的安全处理 |
| 支付凭证 | Payment Voucher | 证明支付操作完成的相关凭据，如银行转账凭证、收据等 | 提供支付证明，支持财务对账和业务审计需求 |
| 退款流水 | Refund Record | 记录支付单每次退款操作的详细信息，包含退款金额、原因、状态等 | 退款操作的完整记录，用于财务对账和业务审计 |
| 对账 | Reconciliation | 与支付渠道核对交易数据的过程，确保系统内交易记录与渠道记录一致 | 保证财务数据准确性，识别和处理交易差异 |
| 已支付金额 | Paid Amount | 支付单已经成功支付并确认的累计金额 | 反映支付进度，用于计算剩余待支付金额和支付完成度 |
| 待支付金额 | Pending Amount | 支付单还需要支付的剩余金额，等于总金额减去已支付金额 | 指导后续支付计划，确定支付单是否完成 |

## 状态枚举术语 (State & Enumeration Terms)

| 中文术语 | 英文术语 | 定义 (Definition) | 业务含义 (Business Meaning) |
|---------|----------|-------------------|---------------------------|
| 未支付 | Unpaid | 支付单刚创建完成，尚未开始任何支付操作的初始状态 | 表示支付义务已确立但未履行，需要客户采取支付行动 |
| 支付中 | Paying | 支付请求已发起并提交到支付渠道，等待渠道处理确认的过程状态 | 表示支付操作正在进行中，需要等待最终结果确认 |
| 部分支付 | Partial Paid | 支付单已收到部分款项，但仍有剩余金额需要继续支付的中间状态 | 支持灵活的分期付款，允许客户根据资金情况分次完成支付 |
| 已支付 | Paid | 支付单所需金额已全部收到，支付义务完全履行的完成状态 | 表示支付义务完全履行，可以推进后续业务流程 |
| 部分退款 | Partial Refunded | 支付单已发生部分金额退款，但仍有剩余已支付金额的状态 | 适应业务变更场景，支持灵活的退款处理 |
| 全额退款 | Full Refunded | 支付单已支付金额已全部退回，恢复到未支付等效状态 | 业务完全取消或重大变更时的资金返还状态 |
| 支付失败 | Failed | 支付过程中出现不可恢复错误，导致支付无法完成的失败状态 | 需要重新发起支付或采用其他支付方式的异常状态 |
| 已停止 | Stopped | 因业务需要主动终止支付操作，暂停支付处理的管控状态 | 支持业务流程控制，避免不必要的支付操作 |
| 已冻结 | Frozen | 因风控或特殊原因暂时限制支付操作，需要人工解除的管控状态 | 保障资金安全，防范异常交易风险的保护性状态 |
| 退款中 | Refunding | 退款请求已发起并提交到支付渠道，等待渠道处理的过程状态 | 表示退款操作正在处理中，需要等待退款结果确认 |
| 退款失败 | Refund Failed | 退款操作在处理过程中出现错误，无法完成退款的失败状态 | 需要重新处理退款或采用其他退款方式的异常状态 |
